Intuitive Approach to Prime Checking

Let's start from the basics: a prime number is only divisible by 1 and itself.

Question 1: If you want to check whether a number n is prime, what is the most straightforward way you might think of?

Answer: Check if any number from 2 to n-1 divides n evenly. If none do, it's prime.

Question 2: That method works, but can we do fewer checks? Think about factors: if a × b = n, what can you say about a and b relative to √n?

Answer: At least one of the factors must be ≤ √n. So we only need to check divisors up to √n.

Question 3: Are there any numbers we can skip checking altogether? For example, after checking 2, what about even numbers?

Answer: After confirming the number is not divisible by 2, we can skip all other even divisors. So we only need to test odd numbers (and 2).

Question 4: Combine these ideas: what is an efficient algorithm to test if n is prime?

Answer:

  1. If n ≤ 1 → not prime.
  2. If n = 2 → prime.
  3. If n is even → not prime.
  4. Check odd divisors from 3 up to √n (inclusive). If any divide n, not prime; else prime.

Now try applying this logic to a few numbers (e.g., 29, 15, 97, 100) and see if you get the correct result.

Remember: The goal is to understand why we check up to √n and skip evens. Try to derive this yourself next time!
